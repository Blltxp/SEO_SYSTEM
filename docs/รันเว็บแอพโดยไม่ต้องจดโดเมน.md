# รัน SEO System เป็นเว็บแอพโดยไม่ต้องจดโดเมน

โปรเจคนี้ **build ได้** แล้ว แต่มีข้อจำกัดเรื่องการ deploy บน Vercel เพราะใช้ **SQLite (better-sqlite3)** และ **Puppeteer** ซึ่ง serverless ไม่รองรับ

---

## ทางเลือกที่เหมาะกับคุณ (ไม่จดโดเมน + ไม่ต้องเปิด IDE/เทอร์มินัลบ่อย)

### 1) รันบนเครื่องตัวเอง — ใช้ได้ที่ `http://localhost:3000`

**วิธีแบบ “เหมือนโปรแกรม” — ดับเบิลคลิกไอคอนแล้วเปิดเบราว์เซอร์เลย**  
1. ไปที่โฟลเดอร์โปรเจค → ไปที่ **`scripts`**  
2. คลิกขวาที่ **`scripts/SEO-System.vbs`** → **ส่งไปที่** → **Desktop (สร้าง shortcut)**  
3. คลิกขวา shortcut บน Desktop → **เปลี่ยนไอคอน** (ถ้าต้องการ) เลือกไอคอนที่ชอบ  
4. ดับเบิลคลิก shortcut นี้ → เบราว์เซอร์จะเปิดไปที่ SEO System โดยอัตโนมัติ (ไม่โชว์หน้าต่าง CMD)  
5. เซิร์ฟเวอร์จะรันอยู่ที่แถบงาน (หน้าต่างย่อ “SEO System - Server”) ถ้าจะปิดแอพให้ปิดหน้าต่างนั้น  

**วิธีที่ 2: ดับเบิลคลิกสคริปต์**  
- ดับเบิลคลิก **`SEO-System.bat`** (ที่โฟลเดอร์รากโปรเจค) หรือ **`scripts/start-seo-system.bat`**  
- เปิดเบราว์เซอร์ให้เอง แล้วหน้าต่าง CMD ปิดเอง  
- เซิร์ฟเวอร์รันอยู่ที่แถบงาน “SEO System - Server” — ปิดหน้าต่างนั้น = ปิดแอพ  

**ทางเลือก: ให้รันอัตโนมัติเมื่อเปิดเครื่อง**  
1. กด `Win + R` พิมพ์ `shell:startup` แล้ว Enter  
2. ลาก shortcut ของ **`SEO-System.bat`** หรือ **`scripts/SEO-System.vbs`** มาวางในโฟลเดอร์นี้  
3. ครั้งต่อไปที่เปิดเครื่อง จะรันให้เอง แล้วเปิดเบราว์เซอร์ไปที่ **http://localhost:3000**

**ถ้าอยากให้เครื่องอื่นในเครือข่ายเดียวกันเข้าได้ (ไม่ต้องจดโดเมน):**  
- ใช้ IP ของเครื่องที่รัน เช่น `http://192.168.1.5:3000`  
- ต้องไม่มี Firewall บล็อก port 3000

---

### 2) ใช้โฮสต์ฟรีที่ให้ subdomain (เช่น xxx.railway.app) — ยังไม่ต้องจดโดเมน

ถ้าอยากให้เข้าได้จากอินเทอร์เน็ต (ไม่ใช่แค่ในเครือข่ายบ้าน):

| บริการ   | Subdomain ฟรี | หมายเหตุ |
|----------|----------------|----------|
| Railway  | ✅ xxx.railway.app | ต้องใช้ PostgreSQL (Neon ฟรี) แทน SQLite และอาจต้องปรับ Puppeteer |
| Render   | ✅ xxx.onrender.com | เหมือนกัน ต้องใช้ DB ภายนอก + จัดการ Puppeteer |
| Fly.io   | ✅ xxx.fly.dev    | มี free tier |

ข้อจำกัด: โปรเจคใช้ **Puppeteer** (เปิด Chrome) ทำให้บนโฮสต์ฟรีต้องใช้บริการอย่าง Browserless หรือปรับเป็น API อื่น จึงต้องแก้โค้ดเพิ่ม

---

## สรุป

- **ไม่จดโดเมน:** ใช้ **localhost** หรือ **IP เครื่อง** (เช่น 192.168.x.x:3000) ก็เพียงพอ  
- **ไม่ต้องเปิด IDE/เทอร์มินัลทุกครั้ง:** ใช้ **`SEO-System.bat`** หรือ **`scripts/start-seo-system.bat`** หรือใส่ shortcut ใน **Startup**  
- **Vercel ไม่เหมาะ** เพราะโปรเจคใช้ SQLite + Puppeteer ซึ่ง serverless ไม่รองรับโดยตรง
