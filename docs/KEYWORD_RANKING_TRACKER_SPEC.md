# Keyword Ranking Tracker — สรุปส่วนที่ต้องมีและทำอะไรได้บ้าง

## เป้าหมาย

ระบบเช็คอันดับ Google ของ **keyword หลัก** ว่า **แต่ละเว็บทั้ง 6 เว็บ** ติดอันดับที่เท่าไหร่ หน้าไหน — เพื่อดูแนวโน้มและเป้าหมาย **ครองหน้าแรก**

### กรณีพิเศษ: keyword 「หาแรงงานต่างด้าว」

- **เวลา search ต้องไม่เจอเว็บของเรา — เพื่อภาพลักษณ์**
- ระบบ Tracker ควร treat keyword นี้ต่างจากคำอื่น: **เป้าหมาย = เว็บเราไม่โผล่ในผลค้นหา** (ไม่ต้องการครองหน้าแรกสำหรับคำนี้)
- ถ้าเช็คแล้วพบว่าเว็บเราไปโผล่ในผลค้นหา → แจ้งเตือนหรือแสดงสถานะพิเศษ (เช่น 「พบเว็บเรา — ควรตรวจสอบ」) แทนการแสดงเป็นอันดับที่ดี

## ข้อจำกัด (ที่ต้องอยู่กับมัน)

- **ไม่ใช้บริการเสียเงิน** (โปรเจคส่วนตัว ไม่มีงบ)
- วิธีที่เป็นไปได้: **scrape ผลค้นหา Google** (เช่น `google.com/search?q=keyword`) แล้ว parse หา URL ของเรา 6 เว็บ ว่าอยู่ลำดับที่เท่าไหร่ หรือใช้ **Google Custom Search API** ในโควต้าฟรี (จำกัดครั้ง/วัน)

## ส่วนที่ต้องมี

### 1. ข้อมูลที่เก็บ (มีโครงอยู่แล้ว)

- **keywords** — 19 คำ (มีใน DB แล้ว หรือรันสคริปต์อัปเดต)
- **sites** — 6 เว็บ (มีใน DB แล้ว)
- **rank_history** — บันทึกแต่ละครั้งที่เช็ค (มีตารางแล้ว)
  - `recorded_date` — วันที่เช็ค
  - `site_slug` — เว็บไหน
  - `keyword` — keyword อะไร
  - `rank` — อันดับ (1, 2, 3, … ถ้าไม่ติดอาจเก็บเป็นค่าใหญ่เช่น 999 หรือแยกฟิลด์ `found`)
  - `url` — หน้าที่ติด (ลิงก์บทความที่ขึ้นในผลค้นหา)

### 2. ฟังก์ชันหลัก (Logic / Automation)

| ฟังก์ชัน | รายละเอียด |
|----------|-------------|
| **เช็คอันดับ 1 keyword** | ส่งคำค้นไป Google (หรือ API) → ดูผล 100 อัน (หรือเท่าที่ได้) → หา URL ของ 6 เว็บเราว่าอยู่ตำแหน่งที่เท่าไหร่ → คืน (site_slug, rank, url) |
| **เช็คทุก keyword พร้อมกัน** | วน 19 keyword เรียกเช็คอันดับ → รวมผลเป็นชุดเดียว (keyword 「หาแรงงานต่างด้าว」ใช้ logic แยก: เป้าหมาย = ไม่ให้เจอเว็บเรา) |
| **บันทึกลง rank_history** | หลังเช็คเสร็จ เขียนลง DB ใช้ `recorded_date` = วันนั้น (หรือ timestamp รอบนั้น) |
| **รันอัตโนมัติ (cron)** | ตั้งเวลาเช่น **ทุกชั่วโมง** รันเช็คแล้วบันทึก |
| **ปุ่มเช็คเอง** | ใน Dashboard มีปุ่ม “เช็คอันดับตอนนี้” เรียก API/สคริปต์ให้รันทันที |

### 3. Dashboard (หน้าเว็บ)

| ส่วน | ทำอะไรได้ |
|------|------------|
| **ตารางอันดับ (Keyword Ranking Table)** | แถว = keyword (12 แถว) หรือแสดงเป็น keyword × site ก็ได้ / คอลัมน์ = 6 เว็บ หรือ “อันดับล่าสุด” ต่อเว็บ — เซลล์แสดง **อันดับ** (เลข) และ **ลิงก์หน้า** (url) |
| **วัน/เวลาที่เช็คล่าสุด** | แสดงว่า “ข้อมูลตารางนี้มาจากเมื่อไหร่” (จาก `recorded_date` ล่าสุด) |
| **ปุ่ม “เช็คอันดับตอนนี้”** | กดแล้วรันการเช็ค (อาจใช้เวลาหน่อย) แล้วรีเฟรชตาราง |
| **Visibility** | ต่อ (เว็บ, keyword): แสดงสถานะ **🟢 Top 10** / **🟡 Top 50** / **🔴 Not Found** ตาม rank |
| **เปรียบเทียบ** | ออปชั่นเลือกดู: **ครั้งล่าสุด** vs **เปรียบเทียบกับครั้งก่อน** หรือ **ย้อนหลัง (เลือกช่วง/ปี)** — แสดงว่าขึ้น/ลงกี่อันดับ |
| **กราฟเปรียบเทียบ** | ดูด้านล่าง |

#### กราฟเปรียบเทียบ (Rank Graph)

- **เปรียบเทียบ 6 เว็บ** สำหรับ keyword นั้นๆ — เลือก 1 keyword แล้วกราฟแสดงแนวโน้มอันดับของทั้ง 6 เว็บในกราฟเดียว (เส้นหรือแท่งต่อเว็บ)
- **แสดงกราฟย้อนหลัง** ได้ตามช่วงเวลาที่เลือก:
  - 24 ชม.
  - 3 วัน
  - 1 อาทิตย์
  - 2 อาทิตย์
  - 3 อาทิตย์
  - 1 เดือน
  - 3 เดือน
  - 6 เดือน
  - 1 ปี

### 4. การเช็คอันดับแบบไม่เสียเงิน (แนวทางเทคนิค)

- **Scrape Google Search**
  - ใช้ `https://www.google.com/search?q=keyword` (อาจต้องใส่ User-Agent / เลี่ยงการถูกบล็อก)
  - Parse HTML หา link ในผลค้นหา (โครง DOM ของ Google เปลี่ยนบ่อย ต้องดูแล)
  - วนดูว่า URL ใดตรงกับโดเมนของเรา 6 เว็บ → ได้ rank กับ url
- **ทางเลือก: Google Custom Search API**
  - โควต้าฟรีจำกัด (100 queries/วัน) — 12 keyword × 1 ครั้ง = 12 queries ต่อรอบ ถ้ารันทุกชั่วโมงอาจเกิน ต้องจำกัดความถี่หรือจำนวน keyword ต่อรอบ

## สิ่งที่ทำได้เมื่อมี Tracker (รวมกับของเดิม)

1. **ดูตารางอันดับล่าสุด** — แต่ละ keyword แต่ละเว็บ ติดอันดับที่เท่าไหร่ หน้าไหน
2. **กดเช็ค keyword** (หรือเช็คทั้ง 12 พร้อมกัน) แล้วเห็นผลทันที
3. **รันเช็คทุกชั่วโมงอัตโนมัติ** + **ปุ่มเช็คเอง**
4. **เปรียบเทียบกับครั้งล่าสุด / ย้อนหลัง** ว่าขึ้นหรือลง
5. **กราฟเปรียบเทียบ** — เลือก keyword แล้วดูกราฟอันดับของ 6 เว็บย้อนหลังได้ (24 ชม. / 3 วัน / 1–3 อาทิตย์ / 1–6 เดือน / 1 ปี)
6. **ส่วน “หัวข้อบทความแนะนำ — เว็บที่ยังไม่อยู่หน้า 1”** จะใช้ข้อมูลจริงจาก `rank_history` (ตอนนี้ใช้แค่ข้อมูลตัวอย่าง)

- การจัดการ keyword/site ใน DB ผ่าน UI (เพิ่ม/ลบ/แก้)

## สรุปสั้น ๆ

| หัวข้อ | รายละเอียด |
|--------|-------------|
| **ต้องมี** | เช็คอันดับ **19 keyword** × 6 เว็บ, บันทึกลง rank_history, ตาราง Dashboard, ปุ่มเช็คเอง, cron ทุกชั่วโมง (หรือตามที่กำหนด), Visibility (Top 10 / Top 50 / Not Found), เปรียบเทียบกับครั้งล่าสุด/ย้อนหลัง, **กราฟเปรียบเทียบ 6 เว็บต่อ keyword** (ย้อนหลัง 24 ชม. / 3 วัน / 1–3 อาทิตย์ / 1–6 เดือน / 1 ปี), **กรณีพิเศษ keyword 「หาแรงงานต่างด้าว」** = เป้าหมายคือไม่ให้เจอเว็บเรา (เพื่อภาพลักษณ์) |
| **ทำได้** | ดูอันดับล่าสุด, เช็คใหม่เมื่อไหร่ก็ได้, ดูแนวโน้มขึ้น/ลงในกราฟ (เลือก keyword + ช่วงเวลา), ให้ส่วน “แนะนำหัวข้อเมื่อยังไม่อยู่หน้า 1” ใช้ข้อมูลจริง |
| **ข้อจำกัด** | ไม่เสียเงิน — ใช้ scrape หรือ Custom Search โควต้าฟรี |
| **มีอยู่แล้ว** | ตาราง keywords, sites, rank_history ใน DB |

ถ้าตรงกับที่คิดไว้ เราค่อยลงมือทำส่วน Tracker (scrape/API + script บันทึก + Dashboard ตาราง + ปุ่ม + cron) ต่อได้เลย。
